## Key decisions

1. **Every new flag defaults off** — v1.0 callers get byte-parity behavior with zero config changes
2. **Plan artifacts stay on parent branch** — `--branch`/`--worktree` creation happens post-plan pre-execute so compose/bounce artifacts remain reviewable before switching branches
3. **Additive / must-not-block for `--live`** — launcher failure logs warning and falls through to inline execution; live mode can never block the runner
4. **Branch/worktree cleanup explicitly deferred** — runner leaves everything intact for human review/merge; automated cleanup is a separate utility for v1.2+
5. **Mutual exclusion for `--branch` and `--worktree`** — parser errors on both set; simpler than negotiating which one wins
