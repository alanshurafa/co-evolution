<objective>

These are planned together because delta tracking's sole consumer is state.json — splitting them would require two rounds of `write_state_*` plumbing.

Purpose: Give Phase 8's eval scorer a single machine-readable file per run. Today the runner writes scattered artifacts (`run.log`, `execute-diffstat.txt`, `verdict.json`, `outputs/bounce-NN.txt`); Phase 8 can't consume a log file as ground truth. state.json consolidates the truth surface: which phases ran, how long each took, what the marker counts converged to, what files changed (delta), and what verdict the verifier returned.

  **Helper 1: `snapshot_workdir_hashes <workdir> <output-manifest-path>`**
  Walks `$workdir`, skipping `.git/`, `runs/`, and any path containing `/.co-evolution/`. Produces JSON of the form `{"path1": "sha256...", "path2": "sha256..."}` (flat object, keys sorted). Uses `find` + `sha256sum`. Paths in keys are relative to `$workdir` with forward slashes.

  ```bash
  snapshot_workdir_hashes() {
    local workdir="${1:?snapshot_workdir_hashes requires a workdir}"
    local output_path="${2:?snapshot_workdir_hashes requires an output path}"
    local tmp_list
    tmp_list=$(mktemp)

    # Walk $workdir, skip .git and runs/ (our own runtime dir).
    # Use null-delimited pipeline to tolerate filenames with spaces/newlines.
    (
      cd "$workdir" && find . -type f \
        -not -path './.git/*' \
        -not -path './runs/*' \
        -not -path '*/.co-evolution/*' \
        -print0
    ) > "$tmp_list"

    if command -v jq >/dev/null 2>&1; then
      # Build JSON via jq: stream of {path, hash} → object.
      (
        cd "$workdir" && \
        xargs -0 -I{} sh -c 'printf "%s\t%s\n" "$(sha256sum "$1" | cut -d" " -f1)" "${1#./}"' _ {} < "$tmp_list" \
          | jq -R -s 'split("\n") | map(select(length>0) | split("\t") | {(.[1]): .[0]}) | add // {}'
      ) > "$output_path"
    else
      # Fallback: printf-based JSON. Assumes filenames contain no quotes or backslashes.
      # Acceptable tradeoff: workdir is a code repo; pathological filenames are out of scope.
      printf '{\n' > "$output_path"
      local first=1
      while IFS= read -r -d '' rel; do
        local hash
        hash=$(cd "$workdir" && sha256sum "$rel" | cut -d' ' -f1)
        local clean_path="${rel#./}"
        if (( first )); then first=0; else printf ',\n' >> "$output_path"; fi
        printf '  "%s": "%s"' "$clean_path" "$hash" >> "$output_path"
      done < "$tmp_list"
      printf '\n}\n' >> "$output_path"
    fi

    rm -f "$tmp_list"
  }
  ```

  **Helper 2: `compute_execute_delta <baseline-json-path> <current-json-path> <output-delta-path>`**

  Reads two manifests, emits `{"modified": [...], "added": [...], "deleted": [...]}` sorted arrays. Use jq when available:
  ```bash
  compute_execute_delta() {
    local baseline="${1:?baseline required}"
    local current="${2:?current required}"
    local output="${3:?output required}"

    if command -v jq >/dev/null 2>&1; then
      jq -n \
        --slurpfile b "$baseline" \
        --slurpfile c "$current" \
        '
        ($b[0] // {}) as $B |
        ($c[0] // {}) as $C |
        {
          modified: [ ($B | keys[]) | select(($C[.] // null) != null and $C[.] != $B[.]) ] | sort,
          added:    [ ($C | keys[]) | select(($B[.] // null) == null) ] | sort,
          deleted:  [ ($B | keys[]) | select(($C[.] // null) == null) ] | sort
        }' > "$output"
    else
      # jq-free fallback: emit empty arrays + log a warning.
      # Rationale: the happy path has jq; the fallback just prevents runner breakage.
      # The eval scorer in Phase 8 can detect the empty-array case and re-hash if needed.
      log "WARNING: jq unavailable — execute_delta fallback produces empty arrays"
      printf '{"modified":[],"added":[],"deleted":[]}\n' > "$output"
    fi
  }
  ```

  **Helper 4: `write_state_phase <state-json-path> <phase-name> <status> <exit-code> <started-at> <completed-at>`**

  Appends a phase entry to `state.phases`. Uses jq if available; falls back to re-emitting the whole file with a naive insert if not.

  ```bash
  write_state_phase() {
    local state_path="${1:?state path required}"
    local phase_name="${2:?phase name required}"
    local status="${3:?status required}"
    local exit_code="${4:?exit code required}"
    local started_at="${5:?started_at required}"
    local completed_at="${6:?completed_at required}"

    if command -v jq >/dev/null 2>&1; then
      local tmp
      tmp=$(mktemp)
      jq --arg name "$phase_name" \
         --arg status "$status" \
         --argjson exit_code "$exit_code" \
         --arg started "$started_at" \
         --arg completed "$completed_at" \
         '.phases += [{
           name: $name,
           started_at: $started,
           completed_at: $completed,
           status: $status,
           exit_code: $exit_code
         }]' "$state_path" > "$tmp" && mv "$tmp" "$state_path"
    else
      log "WARNING: jq unavailable — write_state_phase skipping ($phase_name)"
    fi
  }
  ```

  **Helper 5: `write_state_field <state-json-path> <jq-path> <value-type> <value>`**

  Generic field-setter: `write_state_field state.json '.marker_counts.contested' 'number' 3` or `... '.verify_verdict' 'string' APPROVED` or `... '.execute_delta' 'rawfile' path/to/delta.json`.

  ```bash
  write_state_field() {
    local state_path="${1:?state path required}"
    local jq_path="${2:?jq path required}"
    local value_type="${3:?value type required (string|number|bool|null|rawfile)}"
    local value="${4-}"

    command -v jq >/dev/null 2>&1 || {
      log "WARNING: jq unavailable — write_state_field skipping ($jq_path)"
      return 0
    }

    local tmp
    tmp=$(mktemp)
    case "$value_type" in
      string)
        jq --arg v "$value" "$jq_path = \$v"              "$state_path" > "$tmp" ;;
      number)
        jq --argjson v "$value" "$jq_path = \$v"          "$state_path" > "$tmp" ;;
      bool)
        jq --argjson v "$value" "$jq_path = \$v"          "$state_path" > "$tmp" ;;
      null)
        jq "$jq_path = null"                              "$state_path" > "$tmp" ;;
      rawfile)
        jq --slurpfile v "$value" "$jq_path = \$v[0]"     "$state_path" > "$tmp" ;;
      *)
        rm -f "$tmp"
        die "Unsupported value_type: $value_type"
        ;;
    esac && mv "$tmp" "$state_path"
  }
  ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | T (Tampering) | state.json filesystem write | accept | state.json lives under `$RUN_DIR` (user-owned); an attacker with shell access already has higher-privilege options. No multi-user trust model in this runner. |
| T-07-05 | I (Info disclosure) | baseline_hashes in state.json | accept | SHA-256 hashes leak nothing actionable beyond "this file existed with this content" — a weaker signal than the committed git history already provides. |
| T-07-06 | D (DoS) | snapshot walk on huge workdirs | mitigate | `find` walk excludes `.git/` and `runs/` — the two sources of unbounded file counts. On a typical repo (thousands of files) this is <2s. For pathological cases, per-phase timeout (RNPT-05, Plan 07-03) provides a hard ceiling. |
| T-07-07 | T (Tampering) | jq command injection via task string | mitigate | `init_state_json` passes task via `--arg` (jq) or sed-escapes quotes/backslashes (fallback). Newlines stripped via `tr -d '\r\n'` to prevent JSON-breaking payloads. |
| T-07-08 | I (Info disclosure) | snapshot of sensitive files (e.g. `.env`) | accept | Delta tracking is deliberately unrestricted per CONTEXT.md specifics ("should NOT exclude any paths — the scorer wants truth ground"). state.json stores hashes (not contents), so secret material is not leaked; filename leak is accepted tradeoff. |
