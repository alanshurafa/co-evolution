<objective>
Implement RNPT-05 — the single most painful upstream gap, per UPSTREAM-MESSAGE.md ("one case stuck 1h 39min on a Claude hang"). Every agent invocation must be wrapped in a bash `timeout` with a sensible default (1800s / 30 min), an override flag (`--timeout SECONDS`), and clean state.json accounting when a timeout fires.

Purpose: Prevent the runner from ever hanging indefinitely. A hang today means the operator SIGINTs manually and loses all state; a timeout tomorrow means the run aborts deterministically with a machine-readable record (state.json `.phases[].status == "timeout"`) so evals and retries can handle it.

Output:
- `lib/co-evolution.sh` gains `invoke_agent_with_timeout` wrapper — the public entry for all phase invocations; internally wraps the existing `invoke_agent` dispatcher with `timeout ${PHASE_TIMEOUT}s ...`.
- `dev-review/codex/dev-review.sh` gains `--timeout SECONDS` CLI flag, defaults `PHASE_TIMEOUT=1800`, replaces every `invoke_agent` call site with `invoke_agent_with_timeout`, and records `status="timeout"` + `exit_code=124` in state.json when the wrapper reports a timeout.
- Smoke test: a mock agent that sleeps 5s is killed by a 2s timeout, exit code 124 is captured, state.json shows `status: "timeout"`.
</objective>

- `timeout(1)` returns 124 when it kills the child, 125 if timeout itself failed to start the child, 126/127 for exec errors, or the child's own exit code otherwise.
- Treat 124 as "timeout fired". Treat 125/126/127 as "timeout wrapper broke" (retry bail + error). Treat anything else as the underlying agent's exit code.

  **Item 2: `invoke_agent_with_timeout` wrapper** (at end of file):
  ```bash
  # invoke_agent_with_timeout <agent> <prompt_file> <output_file> <stderr_file> [writable]
  # Same signature as invoke_agent (the dispatcher in dev-review.sh), but wrapped
  # in `timeout(1)`. Sets the global $LAST_INVOKE_EXIT_CODE for the caller to
  # inspect (124 = timeout fired, 0 = ok, other = underlying agent exit code).
  invoke_agent_with_timeout() {
    local agent="${1:?agent required}"
    local prompt_file="${2:?prompt file required}"
    local output_file="${3:?output file required}"
    local stderr_file="${4:?stderr file required}"
    local writable="${5:-false}"

    # Timeout value: caller can override PHASE_TIMEOUT per-call by exporting
    # it before invocation. Default is the library-level default.
    local effective_timeout="${PHASE_TIMEOUT:-1800}"

    # Guard against silly values.
    if ! [[ "$effective_timeout" =~ ^[0-9]+$ ]] || (( effective_timeout < 1 )); then
      die "PHASE_TIMEOUT must be a positive integer (got: $effective_timeout)"
    fi

    if ! command -v timeout >/dev/null 2>&1; then
      # coreutils timeout should be universally available in Git Bash / MINGW64.
      # If missing, log a warning and fall through to the un-timeout-guarded
      # dispatcher. Better to run without timeout than to refuse to run.
      log "WARNING: timeout(1) not found — invoke_agent_with_timeout degrading to direct dispatch"
      invoke_agent "$agent" "$prompt_file" "$output_file" "$stderr_file" "$writable"
      LAST_INVOKE_EXIT_CODE=0
      return 0
    fi

    # Wrap invoke_agent in timeout. Because invoke_agent is a bash function
    # (not an external command), we must invoke it through `bash -c` so that
    # timeout has a real subprocess to kill.
    # We pass the function definitions via export + subshell sourcing, or
    # simpler: re-source lib/co-evolution.sh and the caller's dev-review.sh
    # inside the subshell. Cleanest approach: export the function and re-run.
    #
    # Even simpler: invoke_agent's body is small (5 lines of case/esac). We
    # inline the dispatch here so timeout wraps the actual CLI call, not a
    # bash subshell bootstrap. The duplication is acceptable — invoke_agent
    # is the one-line dispatcher specified by RNPT-01; this wrapper adds
    # the timeout bracket at the exact point the CLI process is spawned.

    local exit_code=0
    case "$agent" in
      codex)
        timeout --foreground "${effective_timeout}s" \
          bash -c 'source "$1"; invoke_codex "$2" "$3" "$4"' _ \
          "${BASH_SOURCE[0]}" "$prompt_file" "$output_file" "$stderr_file" \
          || exit_code=$?
        ;;
      opus)
        timeout --foreground "${effective_timeout}s" \
          bash -c 'source "$1"; invoke_claude "$2" "$3" "$4" "$5"' _ \
          "${BASH_SOURCE[0]}" "$prompt_file" "$output_file" "$stderr_file" "$writable" \
          || exit_code=$?
        ;;
      *)
        die "Unsupported agent: $agent"
        ;;
    esac

    LAST_INVOKE_EXIT_CODE="$exit_code"

    if (( exit_code == 124 )); then
      log "WARNING: agent ${agent} timed out after ${effective_timeout}s (exit 124)"
    fi

    return 0
  }
  ```

  Design rationale (fold into a comment in the code):
  - `--foreground` is required because `timeout` otherwise runs the child in a new process group and SIGTERM handling can miss the real blocked process (e.g. Claude CLI waiting on a network read). `--foreground` leaves signal handling with the invoking shell, so `timeout`'s kill reaches the claude/codex child.
  - Re-sourcing `lib/co-evolution.sh` inside `bash -c` is safer than `export -f invoke_claude invoke_codex` because `export -f` is bash-specific and MINGW64 occasionally does weird things with it.
  - `LAST_INVOKE_EXIT_CODE` is a global (not a return value) because return values in bash are limited to 0-255 and we want to preserve the full exit-code semantics for the caller.
  - The wrapper returns 0 always (unless `die` is called) — exit status flows through the global. This matches the existing `invoke_claude`/`invoke_codex` `|| true` pattern.

  Constraints:
  - MUST NOT change `invoke_agent`'s signature (Plan 07-01's contract).
  - MUST NOT remove or modify existing `invoke_claude`/`invoke_codex`. The wrapper calls them through the re-sourced lib.
  - MUST preserve the `|| true` non-fatal behavior — agent errors are data, not runner crashes.
  - Default `PHASE_TIMEOUT=1800` at the top of `lib/co-evolution.sh` via `:` no-op (source-safe).
  - MUST log a clear timeout message when exit code 124 is observed (grepable for debugging).
  </action>
