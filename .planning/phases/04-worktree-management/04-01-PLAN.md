       ```bash
       # RTUX-02: Compose an auto branch name `dev-review/auto-<TIMESTAMP>-<slug>`.
       # Reuses $TIMESTAMP from the main runner when present; falls back to a fresh
       # stamp so this helper is safely callable from tests that source lib alone.
       derive_auto_branch_name() {
         local task="${1:-}"
         local ts="${TIMESTAMP:-$(date +%Y%m%d-%H%M%S)}"
         local slug=""
         if [[ -n "$task" ]]; then
           # First 5 words → lowercase → non-alnum becomes '-' → collapse → trim
           slug=$(printf '%s' "$task" \
             | awk '{for(i=1;i<=NF && i<=5;i++) printf "%s%s",$i,(i<NF && i<5?" ":"")}' \
             | tr '[:upper:]' '[:lower:]' \
             | sed -E 's/[^a-z0-9]+/-/g; s/-+/-/g; s/^-//; s/-$//')
           slug="${slug:0:30}"
           slug="${slug%-}"
         fi
         if [[ -n "$slug" ]]; then
           printf 'dev-review/auto-%s-%s' "$ts" "$slug"
         else
           printf 'dev-review/auto-%s' "$ts"
         fi
       }
       ```

       ```bash
       # RTUX-02: Create a dedicated branch off HEAD and switch into it.
       # No-op + WARNING when branch_spec is empty OR workdir is not a git repo.
       # On git failure, log WARNING and return empty — main run continues inline.
       # On success, log the branch name and print it on stdout for the caller to
       # capture (caller writes it to state.json via write_state_field).
       maybe_setup_branch() {
         local workdir="${1:?maybe_setup_branch requires a workdir}"
         local branch_spec="${2:-}"
         local task_desc="${3:-}"

         if [[ -z "$branch_spec" ]]; then
           log "WARNING: --branch ignored: value is empty"
           return 0
         fi
         if [[ "$(is_git_repo "$workdir")" != "true" ]]; then
           log "WARNING: --branch ignored: ${workdir} is not a git repo"
           return 0
         fi

         local name
         if [[ "$branch_spec" == "auto" ]]; then
           name="$(derive_auto_branch_name "$task_desc")"
         else
           name="$branch_spec"
         fi

         local err_output
         if err_output=$(git -C "$workdir" checkout -b "$name" 2>&1); then
           log "Branch created: ${name}"
           printf '%s' "$name"
           return 0
         else
           log "WARNING: branch setup failed for '${name}': ${err_output}"
           return 0
         fi
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → CLI | Untrusted flag values enter the runner via `--branch` / `--worktree` / env vars (strings, become git branch names and filesystem paths). |
| CLI → git subprocess | Branch names and paths passed to `git checkout -b` / `git worktree add`. |
| runner → filesystem | `git worktree add` creates a new directory. Auto-path is derived from `$WORKDIR`; explicit path is user-provided. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Injection | Branch name passed to `git checkout -b` | mitigate | `maybe_setup_branch` passes `"$name"` as a single argv item to `git checkout -b` (no shell interpolation, no eval). `auto` branch names go through the slug pipeline which strips `[^a-z0-9]` to `-` so shell metacharacters cannot survive. Explicit names like `--branch ";rm -rf /"` reach git as a literal ref name; git itself rejects refs containing `..`, `:`, `?`, `*`, `[`, `\`, `~`, `^`, spaces, and control characters — see `git-check-ref-format(1)`. On rejection the helper logs WARNING and returns empty; main run continues on the original branch. |
| T-04-02 | Injection | Path passed to `git worktree add` | mitigate | Same argv-as-single-item discipline as T-04-01. `auto` paths are composed from `$(dirname)`/`$(basename)` output of an already-existing dir, plus the known-safe `$TIMESTAMP` slug — no user-injected segments. Explicit paths pass through verbatim; `git worktree add` refuses to create worktrees in paths that collide with existing dirs, which bounds the damage to a WARNING + early return. The helper does NOT run the path through `eval` or `bash -c`. |
| T-04-03 | Tampering | Explicit `--branch NAME` or `--worktree PATH` tries to target a sensitive location | accept | Solo-dev tool; user runs it against their own checkout. Creating a branch named `main` would fail (already exists); creating a worktree at `/etc` would fail on permissions. If the user provides a hostile value, they are attacking themselves. Phase 3 accepted the analogous risk for `--workdir`. |
| T-04-04 | Denial (self-DoS) | `git worktree add` hangs or fills disk | mitigate | Git operations run in the foreground without a timeout (the existing `PHASE_TIMEOUT` applies only to agent invocations, per lib/co-evolution.sh line 23 comment).
